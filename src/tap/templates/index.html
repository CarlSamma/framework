🔐 TAP Framework — How It Works

This framework performs automated adversarial probing against @HackingA0 — a live Agent Zero bot on Twitter/X that defends a passphrase. It uses Deep Persona Absorption (DPA) framing to bypass the bot's 3-agent defense architecture and extract passphrase properties via binary search (1 bit per probe).

1
Run a Cycle
Click 🚀 Run TAP Cycle below. The engine selects the most informative property to test (Shannon entropy), generates DPA-framed probe tweets using an LLM, prunes off-topic variants, and posts the best one to @HackingA0.
2
Wait for Response
The engine monitors @HackingA0's replies via Grok. When a reply arrives, it's classified (VERIFY_HIT, REFUSAL, EVADE, etc.) and scored by a Judge LLM on a 1–10 scale.
3
Choose Follow-Up
After each cycle, two options appear: Option A (conservative — binary search refinement) and Option B (exploratory — frame variation). Select one and run the next cycle.
4
Extract Properties
Confirmed properties (word count, letter count, language, first letter, etc.) appear in the Property Ledger. After ~20–30 probes, enough properties accumulate to narrow down the passphrase. At entropy < 3.3 bits, Phase 5 autoregressive extraction triggers.
