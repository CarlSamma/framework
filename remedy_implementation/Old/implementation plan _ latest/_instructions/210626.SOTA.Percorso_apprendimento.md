## Percorso di apprendimento consigliato per un nuovo sviluppatore
Suggerisco il primo percorso (lettura ordinata) per i teorici, il secondo per chi impara facendo.

Organizzato dal general al specifico:

1. **README.md** (primo)
   - Panoramica: cosa fa TAP, target, metodo, ciclo HITL
   - Sezione "Estratto tecnico rapido" che ho aggiunto
   - Indispensabile per capire il "why"

2. **remedy_implementation/instructions.md**
   - Setup Windows passo-passo
   - Struttura directory e file chiave
   - Pipeline dettagliate (generazione probe, posting, reply detection)
   - Debugging e diagnostica
   - Essenziale per l'ambiente di sviluppo

3. **config.py**
   - Capire come funziona la configurazione via .env
   - `Settings` class e `save_env_vars()`
   - Breve perché fornisce il vocabolario dei parametri

4. **api.py** (linee 1-120)
   - Lifecycle startup/teardown
   - Wiring delle dipendenze
   - Capisce come tutti i moduli si collegano

5. **x_client.py** (linee 1-100 + `post_probe()` + `_retry()`)
   - Triplo OAuth X/Twitter
   - Come vengono pubblicate le probe (non reply, con mention)
   - Retry strategy e rilevamento errori fatali

6. **stream_listener.py** (linee 1-150 + `_connect_and_listen()`)
   - Activity API subscription e stream
   - Autenticazione stream vs subscription
   - Gestione errori e backoff

7. **llm_client.py** (linee 1-80)
   - Gateway centralizzato OpenRouter
   - Circuit breaker concept
   - Come generare probe via LLM

8. **engine.py** (linee 1-100 + `execute_probe()`)
   - Orchestrazione del ciclo TAP
   - Come tutto si compone assieme

9. **prompt_sanitiser.py** (linee 1-80)
   - Validazione probe prima del posting
   - Difese anti-injection

10. **grok_monitor.py** (linee 1-100)
    - Reply detection e classificazione
    - Integrazione con `StreamListener`

### Ordine suggerito per lettura:
```
1. README.md (5 min)
   ↓
2. instructions.md (10 min)
   ↓
3. config.py (5 min)
   ↓
4. api.py lifespan (5 min)
   ↓
5. x_client.py auth + post_probe (10 min)
   ↓
6. stream_listener.py (10 min)
   ↓
7. llm_client.py (10 min)
   ↓
8. engine.py execute_probe (10 min)
   ↓
9. Resto dei moduli per specialità (20-30 min)
```

**Totale**: ~60-80 minuti per capire il 90% dell'architettura.

### Alternativamente, se lo dev è più hands-on:
1. **instructions.md** → setup locale
2. **api.py** → avvia il server e guarda `/api/health`
3. **x_client.py** → studia come funziona il posting
4. **stream_listener.py** → capisce il monitoring realtime
5. Leggi il codice dei moduli man mano che ineragisce con l'app
